### Context
최근 AI 에이전트(AI Agent)는 단순히 텍스트를 생성하는 모델을 넘어, 도구 호출(Tool Calling)과 외부 API 연동을 통해 실질적인 업무를 수행하는 방향으로 진화하고 있습니다. OpenAI, Anthropic(Claude), Google(Gemini) 등 주요 모델 제공사들은 각기 다른 에이전트 구축 인터페이스를 제공하고 있으며, 이를 효과적으로 활용하기 위해 각 모델의 기술적 특징과 API 구조를 이해하는 것이 필수적입니다.

### Core
주요 LLM 제공사별 에이전트 구현 방식 및 특징은 다음과 같습니다.

* **OpenAI (Assistants API)**
    * **특징**: 상태 관리(State Management)를 지원하는 스레드(Threads) 개념을 도입하여 대화 기록 보존이 용이합니다.
    * **도구**: 코드 인터프리터(Code Interpreter), 파일 검색(File Search), 병렬 도구 호출(Parallel Tool Calling)을 기본 지원합니다.
    * **장점**: 개발자 생태계가 가장 크고 에이전트 관리를 위한 고수준 API가 잘 구축되어 있습니다.

* **Claude (Tool Use / Function Calling)**
    * **특징**: 복잡한 도구 사용 시 지시 사항 준수 능력이 매우 뛰어나며, 최근 '컴퓨터 사용(Computer Use)' 기능을 통해 GUI 조작까지 확장되었습니다.
    * **도구**: 프롬프트 캐싱(Prompt Caching) 기능을 통해 반복적인 도구 정의 시 발생하는 비용과 지연 시간을 획기적으로 줄일 수 있습니다.
    * **장점**: 추론 성능이 안정적이며 도구 호출의 정확도가 높아 복잡한 논리 설계에 유리합니다.

* **Gemini (Function Calling & Multimodal)**
    * **특징**: 최대 100만 토큰 이상의 거대한 컨텍스트 창(Context Window)을 제공하여 방대한 양의 데이터를 참조한 에이전트 구축이 가능합니다.
    * **도구**: 텍스트뿐만 아니라 이미지, 영상 등 멀티모달(Multimodal) 입력을 기반으로 한 도구 호출을 지원합니다.
    * **장점**: 구글 워크스페이스(Google Workspace) 및 Vertex AI 환경과의 연동성이 뛰어나며 가성비가 우수합니다.

### Insight
AI 에이전트를 구축할 때는 목적에 맞는 모델 선택이 중요합니다. 상태 유지가 복잡한 워크플로우라면 OpenAI의 Assistants API가 적합하며, 높은 신뢰도와 비용 효율성이 요구되는 구조에서는 Claude의 프롬프트 캐싱과 도구 호출 기능을 고려해야 합니다. 또한 방대한 문서를 실시간으로 분석해야 하는 에이전트의 경우 Gemini의 긴 컨텍스트 창이 강력한 무기가 됩니다. 향후 각 서비스의 API Key를 확보하여 실제 도구 호출을 구현하고, 에이전트의 실행 로그를 분석하며 기술적 완성도를 높여갈 계획입니다.
